Node.jsおよびYarnのインストール​

Node.js​

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 16.15.1
nvm use 16.15.1

Yarn​

curl -sSL https://dl.yarnpkg.com/debian/pubkey.gpg | sudo apt-key add -
echo "deb https://dl.yarnpkg.com/debian/ stable main" | sudo tee /etc/apt/sources.list.d/yarn.list
sudo apt-get update -y
sudo apt-get install yarn -y

ポートを開放​

sudo ufw allow ssh; sudo ufw allow 80; sudo ufw allow 443; sudo ufw enable

ARリポジトリをクローン​

git clone https://github.com/ar-io/ar-io-node.git
cd ~/ar-io-node

.envを設定​

nano .env
  • 次のように設定し、domainmu.zzzのすべての場所を自分のウォレットアドレスに置き換えてください。ARConnectから取得します。
.env
GRAPHQL_HOST=arweave.net
GRAPHQL_PORT=443
START_HEIGHT=1000000
ARNS_ROOT_HOST=domainmu.zzz
ADMIN_API_KEY=Password123
AR_IO_WALLET=Address
  • 保存してください：CTRl+X Y Enter

ドメインを設定​

** ドメインマネージャーにアクセスし、どこで購入するかを選択してください。ドメインが他のプロジェクトや個人用のウェブサイトで使用されていないことを確認してください。

  1. すべてのDNSレコードがある場合は削除します。

  2. IP VPS に向かせるAレコードを設定し、@と名前を付けます。

  3. IP VPS に向かせるAレコードを設定し、*と名前を付けます。

SSLを設定​

EmailMU@gmail.comを自分のメールアドレスに置き換え、domainmu.zzzをすべて自分のドメインに置き換えます。

sudo certbot certonly --manual --preferred-challenges dns --email EmailMu@gmail.com -d domainmu.zzz -d '*.domainmu.zzz'

** このステップでは、TXTレコードの設定方法についての指示が表示されます。指示に従って設定してください。

  • それをコピーして、Enter/Continueボタンを押さないでください。
  • ドメインマネージャーに移動して、DNSレコードを追加します。
  • TXTを選択します。
  • 名前に_acme-challengeを入力します。
  • 値にVPS

でコピーしたものを入力します。

  • 次に、https://dnschecker.org/#TXT/に移動します。
  • すべてがチェックマークが付いており、内容が正しい場合は、VPSに戻り、続行してください。

Nginxを設定​

rm /etc/nginx/sites-available/default
nano /etc/nginx/sites-available/default
備考

以下のように設定し、すべてのdomainmu.zzzを自分のドメインに置き換えます。

default
# HTTPからHTTPSへの強制リダイレクト
server {
    listen 80;
    listen [::]:80;
    server_name domainmu.zzz *.domainmu.zzz;

    location / {
        return 301 https://$host$request_uri;
    }
}

# トラフィックをあなたのノードにフォワードし、SSL証明書を提供します
server {
    listen 443 ssl;
    listen [::]:443 ssl;
    server_name domainmu.zzz *.domainmu.zzz;

    ssl_certificate /etc/letsencrypt/live/domainmu.zzz/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/domainmu.zzz/privkey.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_http_version 1.1;
    }
}
  • 保存してください：CTRl+X Y Enter

Nginxを確認​

sudo nginx -t

Nginxを再起動して再度確認​

sudo service nginx restart
sudo nginx -t

OK!!

ノードのセットアップ​

cd ~/ar-io-node
sudo docker compose up -d --build

完了までお待ちください...........

次に、実行中かどうかを確認​

  • ブラウザで次を開きます：IP.VPS.MUをIP VPS MUのIPアドレスに置き換えます。
url
http://IP.VPS.MU:3000/4GnL_uFIAcuiE_w4nPWvg_Z2eHkLnnnNDufPkNwab5M
  • これが表示されたら、成功です！

今後のアップデートがある場合​

ノードを停止​

cd ~/ar-io-node
sudo docker compose down

リポジトリを更新​

git pull

イメージを更新​

sudo docker compose pull

再ビルド​

sudo docker compose up -d --build

Discordでテストトークンをリクエスト​

  • Discordに参加します。
  • #testnetチャンネルに移動します。
  • /applyコマンドを使用します。
  • アンケートに記入し、配布を待ちます

テストトークンを取得したら

リポジトリのコピー​

cd
git clone https://github.com/ar-io/testnet-contract.git
cd ~/testnet-contract
yarn install
yarn build

ARConnect拡張ウォレットを開きます。​

  • エクスポートして、key.jsonという名前を付けます。

  • ファイル key.jsonをVPSに移動し、/testnet-contractディレクトリに移動します。

.env ファイルを作成 (Optional)​

nano .env
  • これで埋めます JWK はあなたの key.json ファイルの内容ですが、それを1行に編集してください
.env
JWK=
ARNS_CONTRACT_TX_ID=bLAgYxAdX2Ry-nt6aH2ixgvJXbpsEYm28NgJgyqfs-U

join-network.tsファイルを編集​

nano tools/join-network.ts
  1. qty = 10_000
  2. label = あなたのニックネーム
  3. fqdn = あなたのドメイン
  4. note = メモ、自由に

  • 保存してください：`

CTRl+X Y Enter`

  • このコマンドを実行します。
yarn ts-node tools/join-network.ts
